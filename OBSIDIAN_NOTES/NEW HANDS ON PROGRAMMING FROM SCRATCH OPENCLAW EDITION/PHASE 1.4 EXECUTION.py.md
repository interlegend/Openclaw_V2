### 🎓 BEGINNER EXPLANATION: Heuristics vs Determinism

In software engineering, there are two ways to check if something worked:

1. **Deterministic (The Right Way):** The OS guarantees that if a command succeeds, it returns the number 0 (called an Exit Code). If it fails, it returns 1 or higher. This is math. It never lies.
    
2. **Heuristic (The AI Slop Way):** Reading the text and looking for the word "OK". This is guessing. It is prone to "False Positives" (thinking it succeeded when it didn't) and "False Negatives" (thinking it failed when it didn't).

```
def run_command(cmd: str, timeout: int = 120) -> dict:
    # 1. Check blacklist. If True, return {"output": "bruh...", "success": False, "needs_sudo": False}
    
    # 2. Setup env (Keep the PATH logic the AI wrote, it was good!)
    # env = os.environ.copy() ... etc.
    
    # 3. Try block with subprocess.run
    try:
        result = subprocess.run(cmd, shell=True, capture_output=True, text=True, timeout=timeout, env=env)
        
        # 4. Merge pipes!
        combined_output = result.stdout.strip()
        if result.stderr.strip():
            combined_output += f"\n[stderr]: {result.stderr.strip()}"
        
        if not combined_output:
            combined_output = "(command returned no output)"
            
        # 5. THE DETERMINISTIC FIX
        # Check if the text contains sudo indicators
        output_lower = combined_output.lower()
        needs_sudo = any(ind in output_lower for ind in SUDO_INDICATORS)
        
        # Check success using MATH, not text guessing!
        is_success = (result.returncode == 0)
        
        return {
            "output": combined_output,
            "success": is_success,
            "needs_sudo": needs_sudo
        }
        
    except subprocess.TimeoutExpired:
        return {"output": "Timeout", "success": False, "needs_sudo": False}
    except Exception as e:
        return {"output": str(e), "success": False, "needs_sudo": False}
```
